# LLMとは

ChatGPT を使ったことはありますか。質問を入れると、まるで人間のように答えが返ってくる、あの体験。実はその裏側で動いているのが **LLM**（**Large Language Model**、**大規模言語モデル**）と呼ばれる技術です。

このレッスンでは、LLM とはいったい何者なのか、何ができて何が苦手なのかを、自分の言葉で説明できるところまで一緒に学んでいきましょう。生成AIパスポート試験では、**LLM の定義・基本動作・ハルシネーション** の三点が頻出です。

---

## このレッスンでできるようになること

- LLM の正式名称と、それが何の略であるかを説明できる
- LLM が「次のトークンを確率的に予測している」という基本動作を説明できる
- 「大規模」が指す二つの大きさ（学習データ・パラメータ数）を説明できる
- ハルシネーションという現象を、自分の言葉で説明できる
- LLM と検索エンジンの違いを区別できる

## 1. LLM の定義

**LLM** は **Large Language Model** の頭文字をとった略語で、日本語では「**大規模言語モデル**」と訳されます。

言葉を分解すると次のようになります。

| 語 | 意味 |
|---|---|
| **Large（大規模）** | 扱うデータの量と、内部の規模が、これまでにないほど大きいこと |
| **Language Model（言語モデル）** | 言葉のつながり方を予測する仕組み |

つまり LLM とは、**膨大な文章を学習して、次にどんな単語が来るかを予測する大きな仕組み** のことです。

少し意外かもしれません。ChatGPT のような対話型 AI は、まるで人間のように考えているように見えます。でも内部では、「考えている」のではなく **「次の単語をひたすら予測している」** だけなのです。

代表的な LLM には、OpenAI の **GPT** シリーズ、Anthropic の **Claude**、Google の **Gemini**、Meta の **Llama** などがあります。提供企業もモデルの規模もさまざまですが、「次の単語を予測する」という基本構造はどれも共通です。

### この節の要点

- **LLM = Large Language Model = 大規模言語モデル**
- 内部では「考えている」のではなく「**次の単語を予測している**」
- 代表例: GPT、Claude、Gemini、Llama

---

## 2. 仕組みのざっくり

では、LLM はどうやって次の単語を予測しているのでしょうか。

たとえば「今日の天気は」という文を入れたとします。人間ならその次に「晴れです」とか「雨です」と続けますよね。LLM も同じことをしていますが、「晴れです」と一本道で決めているわけではありません。内部では、可能性のある単語をたくさんリストアップして、それぞれに **確率** を割り振っています。

| 候補 | 確率 |
|---|---:|
| 晴れ | 40% |
| 雨 | 20% |
| 曇り | 15% |
| 雪 | 5% |
| その他 | 20% |

この確率にしたがって、一つの単語を選びます。選んだ単語を文末につけ足して、また次の単語を予測する。これを延々と繰り返しているのが、LLM の基本動作です。

### トークンという考え方

ここで一つ、専門用語を覚えておきましょう。LLM は、文章をそのままの形で扱っているわけではありません。文章を小さな単位に分割してから内部で処理しています。この単位のことを **トークン**（token）と呼びます。

- 英語のトークンは、だいたい単語ひとつに近い粒度（例: `LLM` は 1 トークン、`unbelievable` は 2〜3 トークンに分割されることが多い）
- 日本語のトークンは、ひらがな1文字〜数文字のかたまり（例: `今日` は 1 トークン、`の` は 1 トークン）

LLM が「次の単語を予測する」と説明される動作は、正確には「次の **トークン** を予測する」動作です。

```mermaid
flowchart LR
  A[入力: 今日の天気は] --> B[トークン化]
  B --> C[確率分布を計算]
  C --> D[トークンを1つ選ぶ]
  D --> E[文末に追加]
  E --> C
```

### この節の要点

- LLM は次に来る単語の **確率分布** を計算し、その中から一つを選ぶ
- 選んだ単語を文末に足して、また次を予測する、を繰り返す
- 処理の最小単位は「**トークン**」と呼ばれる

## 3. 学習データとパラメータ規模

「**大規模**」というキーワードに戻ります。何が大規模なのか、大きく二つあります。

### (1) 学習データの量

最新の LLM は、インターネット上の文章、書籍、論文、ソースコードなど、人間が一生かけても読みきれない量の文章を読み込んで学習しています。ざっと **数兆語（数兆トークン）** というスケールです。

学習に使うデータが多いほど、より幅広い話題に対応でき、より自然な文章を生み出せるようになります。

### (2) パラメータの数

**パラメータ** とは、LLM が学習を通じて少しずつ調整していく、内部の重みのことです。脳でいえば、神経どうしのつながりの強さに近いイメージです。

このパラメータ数が桁違いに増えてきました。

| 世代 | パラメータ数の目安 |
|---|---:|
| 2018 年ごろの BERT 系 | 約 1 億 |
| 2020 年 GPT-3 | 約 1,750 億 |
| 2023 年以降の主要モデル | 数千億〜数兆 |

データ量と内部規模の両方が桁違いに大きくなったことで、LLM はまるで人間が書いたような自然な文章を生み出せるようになりました。これが「大規模」言語モデルと呼ばれる理由です。

### この節の要点

- 「大規模」が指すのは「**学習データの量**」と「**パラメータ数**」の二つ
- 学習データは数兆語スケール
- パラメータ数は初期の **1 億規模** から、現在は **数千億〜数兆** に達している

## 4. できること、苦手なこと

LLM は何ができて、何が苦手なのでしょうか。

### 得意なこと

- 文章を書く（要約・翻訳・メール下書き・コード生成 など）
- 質問への回答、対話形式での受け答え
- 文章のスタイル変換（ですます調 ↔ 砕けた口調 など）
- アイデア出し、文章の校正

言葉を扱うタスクは、ほぼ何でもこなせる水準にあります。

### 苦手なこと: ハルシネーション

代表的な苦手分野が **ハルシネーション**（hallucination、幻覚）と呼ばれる現象です。

> **ハルシネーション** — LLM が、もっともらしい嘘の情報を自信たっぷりに答えてしまう現象。

たとえば、実在しない論文のタイトルを作り出したり、存在しない人物の経歴を語ったり、間違った歴史的事実を断言したりします。LLM 自身には「嘘をついている」という自覚がありません。なぜなら、繰り返しになりますが、LLM は **「次に来そうな単語を予測している」だけ** だからです。「事実かどうか」を判断する仕組みは、もともと持っていません。

そのため、LLM が出した情報は **最終的に人間が事実を確認する** ことが必須になります。これが、LLM を安全に使うための、いちばん大切なルールです。

### この節の要点

- 得意: 文章生成、要約、翻訳、対話、コード生成、アイデア出し
- 苦手: **事実の正確さの保証**（ハルシネーションが起こりうる）
- 利用者は出力結果を **必ず人間が検証** する必要がある

---

## 5. LLM と検索エンジンの違い

試験で混同を狙う選択肢として、「LLM = 検索エンジン」という誤解がよく出題されます。両者は根本的に違う仕組みです。

| 観点 | 検索エンジン（Google など） | LLM |
|---|---|---|
| 出力 | すでに **存在する** Web ページを見つけて並べる | その場で **新しい文章を生成** する |
| 仕組み | インデックスからキーワード一致を探す | 次のトークンを **確率的に予測** する |
| 情報源の追跡 | 結果に URL が付くので **出典が辿れる** | 出典がない（後付けの RAG 構成を除く） |
| ハルシネーション | 基本的に起きない（取得元が実在する） | **起こりうる** |
| 最新情報 | 即時にインデックス更新される | 学習時点までの情報に縛られる |

検索エンジンは「既にあるもの」を見つけてくる仕組み。LLM は「その場で新しく作り出す」仕組み。この区別が、生成AI を理解する出発点になります。

### この節の要点

- 検索エンジン = **既存情報を探して並べる**
- LLM = **その場で新しく文章を作る**
- 出典追跡・最新性・ハルシネーション耐性は **検索エンジンの方が強い**

---

## 6. 試験で問われやすいポイント

生成AIパスポート試験で押さえておきたいのは、次の三点です。

1. **LLM の正式名称** — Large Language Model（大規模言語モデル）。学習データとパラメータの両方が桁違いに大きい。
2. **基本動作** — 次のトークンを **確率的に予測** すること。意味を理解しているわけではなく、確率にしたがって単語を選んでいる、という点が重要です。
3. **ハルシネーション** — もっともらしい誤った情報を生成する現象。利用者は出力結果を必ず検証する必要がある。

「LLM = 単なる検索エンジン」と混同させる選択肢に注意してください。**検索エンジンは既存情報を探す仕組み、LLM はその場で新しい文章を作る仕組み**、というふうに区別できれば落とし穴を回避できます。

### この節の要点

- 出やすい論点 3 つ: **正式名称 / 基本動作 / ハルシネーション**
- 混同を狙う選択肢として「LLM = 検索エンジン」が典型
- 「既存情報を探す」vs「新しく作る」で区別する

## まとめ

このレッスンで学んだことを振り返ります。

- LLM は **Large Language Model**（大規模言語モデル）の略

- 「大規模」が指すのは **学習データの量** と **パラメータ数** の二つ

## 主要用語まとめ

| 用語 | 意味 |
|---|---|
| **LLM** | Large Language Model の略。大規模言語モデル |
| **言語モデル** | 言葉のつながり方を予測する仕組み |
| **トークン** | LLM が文章を処理するときの最小単位。英語は単語に近く、日本語は 1〜数文字 |
| **パラメータ** | LLM が学習で調整する内部の重み。神経のつながりの強さに近い |
| **ハルシネーション** | LLM がもっともらしい嘘の情報を自信たっぷりに答える現象 |
| **GPT / Claude / Gemini / Llama** | 代表的な LLM の例。提供企業はそれぞれ異なる |
