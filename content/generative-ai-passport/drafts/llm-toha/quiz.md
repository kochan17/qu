# 確認問題: LLMとは

`lesson-body.md` の理解を確認するための 5 問。生成AIパスポート試験形式（4 択単一選択）に揃えてある。

各問題には:
- **問題文**
- **選択肢**（`- [x]` が正解）
- **解説**（なぜ正解か / なぜ他が不正解か）
- **難易度**（easy / medium / hard）
- **対応セクション**（lesson-body のどこを問うているか）

---

## Q1. LLM の正式名称

**問題**: LLM の正式名称として正しいものはどれですか。

**選択肢**:
- [ ] Linear Language Model
- [ ] Logical Learning Machine
- [x] Large Language Model
- [ ] Limited Language Module

**解説**:
- 正解は **Large Language Model**（大規模言語モデル）です。
- 「Large（大規模）」は扱うデータの量と内部規模の大きさを、「Language Model（言語モデル）」は言葉のつながり方を予測する仕組みを指します。
- 似た略称や紛らわしい単語に惑わされないよう、正式名称をセットで覚えておきましょう。

**難易度**: easy
**対応セクション**: 1. LLM の定義

---

## Q2. LLM の基本動作

**問題**: LLM が文章を生成するときの基本動作として、最も適切なものはどれですか。

**選択肢**:
- [ ] あらかじめ用意された定型文の中から、文脈に合うものを検索して返している
- [x] 次に来るトークンを確率的に予測し、選んだトークンを足してまた次を予測する、を繰り返している
- [ ] 文章の意味を完全に理解した上で、論理的に正しい文を組み立てている
- [ ] 過去の会話履歴をすべて記憶し、それを再構成して回答している

**解説**:
- 正解は 2 番目の選択肢です。LLM は **「次のトークンを確率的に予測する」** 動作をひたすら繰り返すことで文章を生成しています。
- 1 番目は検索エンジンの説明に近く、LLM とは異なります。
- 3 番目は誤りです。LLM は意味を「理解」しているわけではなく、確率分布から選んでいるだけです。だからこそハルシネーションが起こります。
- 4 番目は誤りです。LLM はセッションごとに与えられたコンテキストを参照しますが、過去の会話を恒久的に記憶しているわけではありません。

**難易度**: easy
**対応セクション**: 2. 仕組みのざっくり

---

## Q3. 「大規模」が指す二つの規模

**問題**: LLM の「大規模（Large）」が指している規模として、適切な組み合わせはどれですか。

**選択肢**:
- [ ] 利用者数と稼働時間
- [ ] サーバー台数と通信速度
- [x] 学習データの量とパラメータの数
- [ ] 開発期間と開発者の人数

**解説**:
- 正解は **学習データの量とパラメータの数** です。
- 学習データは数兆語スケールの文章を読み込んでおり、パラメータ（内部の重み）は数千億〜数兆個に達しています。
- この二つが桁違いに大きくなったことで、人間のように自然な文章を生成できるようになりました。
- 他の選択肢はサービス運営や開発体制の話で、モデル自体の「大規模」とは関係ありません。

**難易度**: medium
**対応セクション**: 3. 学習データとパラメータ規模

---

## Q4. ハルシネーション

**問題**: ハルシネーションに関する説明として、最も適切なものはどれですか。

**選択肢**:
- [ ] LLM が学習データに含まれない最新情報を、リアルタイムでネット検索して提供する機能のこと
- [x] LLM が、もっともらしい嘘の情報を自信たっぷりに答えてしまう現象のこと
- [ ] LLM が処理可能なトークン数の上限を超えてエラーになる現象のこと
- [ ] LLM が同じ質問に対して毎回異なる回答を返すことで、再現性が損なわれる現象のこと

**解説**:
- 正解は **「もっともらしい嘘の情報を自信たっぷりに答えてしまう現象」** です。
- 実在しない論文タイトル・人物・出来事を、あたかも事実のように生成してしまうことを指します。
- LLM は「次に来そうな単語を予測している」だけで、「事実かどうか」を判断する仕組みは持っていません。
- そのため、LLM の出力は **必ず人間が事実を確認** する運用が安全に使うための前提になります。
- 1 番目はネット検索機能、3 番目はトークン制限、4 番目はランダム性の話で、いずれもハルシネーションとは別の概念です。

**難易度**: medium
**対応セクション**: 4. できること、苦手なこと

---

## Q5. LLM と検索エンジンの違い

**問題**: LLM と検索エンジンの違いの説明として、最も適切なものはどれですか。

**選択肢**:
- [ ] LLM も検索エンジンも、どちらも既存の Web ページから情報を取得して並べる仕組みである
- [ ] LLM は出典 URL を必ず返すが、検索エンジンは要約のみを返す
- [x] 検索エンジンは既存情報を探して並べる仕組み、LLM はその場で新しい文章を生成する仕組みである
- [ ] LLM と検索エンジンはどちらもハルシネーションを高い頻度で起こす

**解説**:
- 正解は **「検索エンジンは既存情報を探して並べる仕組み、LLM はその場で新しい文章を生成する仕組み」** です。
- 試験では「LLM = 検索エンジン」と混同を狙う選択肢がよく出ます。両者の役割の違いをしっかり区別できるようにしておきましょう。
- 1 番目は誤りです。LLM は既存ページを取得しているのではなく、確率分布から新しく文章を作っています。
- 2 番目は逆の説明になっています。出典 URL を返すのは検索エンジンで、LLM は基本的に出典を返しません。
- 4 番目は誤りです。ハルシネーションは LLM 特有の現象であり、検索エンジンでは基本的に起きません（取得元が実在するため）。

**難易度**: medium
**対応セクション**: 5. LLM と検索エンジンの違い
