画面構成の基本ルール

動画の1カットは「見出し」「図」「補助説明」の3点に絞ります。余白は情報量を減らすためではなく、どこを見ればよいかを明確にするために使います。

RULE 01

見出しは左上、短く

H1は1行、長くても2行。引用符や長い修飾語で不自然に折り返さない。

RULE 02

図は中央から上

学習者が最初に見る図は画面の中央から上に置く。下端はプレイヤーUIに隠れる前提で避ける。

RULE 03

1カット1メッセージ

同じ画面で定義、例、注意点を全部出さない。順番に見せて認知負荷を下げる。

概念説明の標準構成

N進数、ビット、データベースの正規化、ネットワーク階層など「まず概念をつかむ」教材はこの型を使います。

0-5s

1. ひとことで定義

最初に「これは何の話か」だけを言う。細かい説明はまだ出さない。

例: N進数 = 数え方のルール
5-12s

2. 身近な比喩かモデル

抽象語を図に置き換える。N進数ならスイッチ、正規化なら整理棚のように扱う。

スイッチのON/OFF、カード、数直線
12-22s

3. ルールを1つずつ見せる

桁の重み、繰り上がり、記号の種類など、理解に必要なルールを分割する。

右から 1, 2, 4, 8 の桁カード
22-32s

4. 小さい具体例で確認

いきなり大きな数にしない。2進数 1011 = 11 のように、見て足せる例にする。

8 + 0 + 2 + 1 = 11
32-40s

5. 同じ量を別表記で並べる

最後に「表記が違うだけ」という感覚を作る。次の教材への橋渡しもここで行う。

17 = 10001 = 11

話題別テンプレート

教材の種類によって、同じ動画構成を使い回さない。自然言語で依頼するときも、まずこの型を選びます。

概念理解

N進数、ビット、正規化、OSI参照モデル

抽象概念を最初に理解させたいとき。

  1. ひとことで定義
  2. 身近なモデル
  3. 構成要素
  4. 小さい例
  5. まとめと次の問い
手順理解

基数変換、SQL実行順、探索アルゴリズム

順番通りに操作すれば解けるもの。

  1. ゴールを見せる
  2. 入力と出力を分ける
  3. 手順を3から5個に区切る
  4. 1問だけ通しで解く
  5. チェック問題へつなぐ
比較理解

2進/10進/16進、TCP/UDP、暗号方式

似たものの違いを整理したいとき。

  1. 比較軸を先に出す
  2. 横並びで同じ順番に説明
  3. 違いを色で強調
  4. 使い分けを一言で示す
  5. 誤解しやすい点を潰す
暗記整理

頻出用語、法律、セキュリティ分類

理解よりも分類と想起が重要なもの。

  1. 全体マップを出す
  2. 3から5個にチャンク化
  3. 語呂ではなく意味で結ぶ
  4. 空欄で思い出させる
  5. 確認問題へつなぐ

制作ルール

AIに自然言語で作らせる前提なら、守るべき制約を先に明文化しておく必要があります。

画面サイズ 16:9、1920x1080。重要要素は上下左右8%の安全領域内に置く。
見出し H1は14文字前後を目安にする。長い説明文を見出しに入れない。
図の位置 主役の図は画面中央から上に置く。下16%は動画コントロールの被りを想定して空ける。
文字量 1カットあたり本文は2行まで。読ませる文章はテキスト教材に寄せる。
色 背景は明るく、強調色は2色まで。黒い大面積背景は避け、必要な場合だけ小さく使う。
確認 レンダリング前に、初期フレーム、各シーン中盤、最終フレームのスクリーンショットを確認する。

OK

  • タイトル、図、補助説明の位置を固定する
  • 図だけ見ても意味が追えるようにする
  • 細かい文章はテキスト教材へ逃がす
  • 動画では理解の入口と流れを担当する

NG

  • 見出しが変な位置で改行される
  • 主役の図が画面下部に寄りすぎる
  • 1画面に定義、例、注意点を全部入れる
  • 黒背景や強い装飾で教材内容よりUIが目立つ
