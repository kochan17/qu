> div (tc-curriculum-mcp/curriculum) のカリキュラム構造を踏襲し、Qu のターゲット（20代女性、iPad中心、生成AI初心者）に合わせて翻訳したもの。

## 1. ファイル配置と命名

```

├── lesson-body.md      ← この雛形で書く（正典）
├── script.md           ← lesson-body から派生する音声/動画ナレーション台本
├── quiz.md             ← lesson-body から派生する確認問題
└── video/              ← HyperFrames プロジェクト（必要時）
```

lesson-slug: kebab-case 短く意味が通る形（例: `llm-toha`、`transformer`、`prompt-design`）

## 4. 文体ルール

### トーン

- **「です・ます調」基本**。崩しすぎない、堅すぎない
- **アスピレーショナル形**（提案・呼びかけ・「〜していきましょう」「〜できるようになります」）
- ❌ 断言（「絶対」「必ず」「100%」）
- ❌ 煽り・FOMO（「今やらないと損」「合格できない」）
- ❌ 命令形（「覚えてください」「やってください」）
- ✅ 「〜ですね」「〜でしょうか」を 1ブロックに 1〜2回 まで（対話感）
- ✅ 「カフェで一緒に勉強している先輩」のような寄り添い

### 専門用語の扱い（最重要）

- **初出は必ず定義** する
  - 例: 「**LLM**（Large Language Model、大規模言語モデル）とは、〜」
- **略語は正式名称併記**（LLM = Large Language Model など）
- **2回目以降は普通に使う**（毎回定義し直さない）
- **見出し (h1/h2) に未定義の専門用語を入れない**

### 文の長さ

- 1文 60〜80字以内が目安
- 「〜と思います」「〜だと言えるでしょう」のような **不要な緩衝表現は削る**
- ら抜き言葉禁止（「出れる」→「出られる」）

### 太字 (`**...**`) の使い方

div の教材は **重要語を惜しまず太字** にする。Qu も同様に:

- 初出の専門用語は太字で目立たせる
- 章の核となるキーワードは太字
- 1段落に太字 1〜3 個が目安、太字だらけにしない

### 数字・記号

- 数字は半角でも全角でも、**章内で統一**（迷ったら半角）
- パーセントは「40%」または「40 パーセント」、章内で統一
- 大きな数字は **桁区切りカンマ**（1,750 億）
- 「〜年に」「〜倍」のような単位は数字に隣接

---

## 5. 画像・図解の方針

### 密度

- **3〜5 行のテキストごとに画像 or 図解 or 表 or コードブロック** を挟む
- 6 行以上連続するテキストブロックは見直し対象

### 画像の種類

| タイプ | 使いどころ | 例 |
|---|---|---|
| Mermaid 図 | 概念のフロー・関係性 | `flowchart LR ...` |
| Markdown 表 | 比較・分類 | 「LLM vs 検索エンジン」 |
| 引用ブロック | 定義の強調 | `> **ハルシネーション** — ...` |
| コードブロック | コード例（資格学習では稀） | プロンプト例など |
| HyperFrames 短尺 GIF/MP4 | 動きで見せたい概念 | （将来実装） |

### 図のプレースホルダ

HyperFrames で図解動画/GIF を後から差し込む前提なら、本文中に:

```markdown
![{図のタイトル：トークン分割の流れ}](TBD-hyperframes-clip)
```

のように TBD プレースホルダを書いておく（後で差し替え）。

## 6. 見出しスタイル

| 階層 | フォーマット | 例 |
|---|---|---|
| `# {title}` | 体言止め | 「LLMとは」 |
| `## 1. {セクション名}` | 体言止め or「〜とは」 | 「LLM の定義」「仕組みのざっくり」 |
| `### この節の要点` | 固定フレーズ | （固定）|
| `### {小見出し}` | 体言止め or 「〜について」 | 「トークンという考え方」 |

「〜について知ろう」「〜を学ぼう」は div 標準だが、Qu トーンではやや堅い。
**使ってもいいが多用しない**。基本は体言止めで揃える。

---

## 7. 長さの目安

| 項目 | 目安 |
|---|---|
| 全体文字数 | 3,000〜5,500 字 |
| 読了時間 | 7〜10 分 |
| セクション数 | 5〜7 |
| 1セクションの文字数 | 400〜800 字 |
| 1段落の行数 | 3〜5 行 |

---

## 8. アンチパターン

書いてはいけない例:

| ❌ NG | ✅ OK |
|---|---|
| 「絶対に出題されます！」 | 「試験で問われやすいポイントです」 |
| 「あなたは知らないかもしれませんが」 | 「ご存じない方もいらっしゃるかもしれません」 |
| 「以下を覚えてください」 | 「以下を押さえておきましょう」 |
| 「重要！！！」 | 「ここがポイントです」 |
| 「※注意：めっちゃ大事」 | 「ここは少し注意が必要です」 |
| 数字なし「沢山」「多い」 | 数字つき「1,750億個」「数兆語」 |
| 未定義の略語をいきなり使う | 初出で「LLM（Large Language Model）」 |

## 9. 公開前チェックリスト

`status: published` に切り替える前に通す:

### 構成
- [ ] frontmatter が完備されているか（id / certification / title / level / reading_time_minutes / tags / status）
- [ ] 「はじめに」「このレッスンでできるようになること」「まとめ」「主要用語まとめ」「次に学ぶこと」「参考リンク」が揃っているか
- [ ] 各セクションに「この節の要点」があるか
- [ ] セクション数が 5〜7、全体 3,000〜5,500 字に収まっているか

### 文章
- [ ] 初出の専門用語に **必ず定義** が付いているか
- [ ] 略語が **正式名称併記** されているか
- [ ] 見出し (h1/h2/h3) に未定義の専門用語がないか
- [ ] 断言・煽り・命令形・FOMO 表現がないか
- [ ] ら抜き言葉・「〜と思います」「〜だと言えるでしょう」が混入していないか

### 視覚
- [ ] 3〜5 行ごとに画像/図/表/引用ブロック等が挿入されているか
- [ ] 6 行以上連続する裸テキストブロックが残っていないか
- [ ] 太字 (`**...**`) が重要語に適切に付与されているか

### 横展開
- [ ] [文章スタイル総則](../../../div/tc-docbase-mcp/02_writing-documentation/01-style-principles.md)（既知→未知 / 解釈ぶれ最小 / 万人比喩）を満たしているか
- [ ] [完全習得学習の基準](../../../div/tc-docbase-mcp/06_learning-design/01-mastery-learning.md)（中学生で説明できる）を満たしているか

### 倫理
- [ ] [Humane Tech Rubric](humane-tech-rubric.md) の Four Promises を通過しているか（13点未満は差し戻し）
