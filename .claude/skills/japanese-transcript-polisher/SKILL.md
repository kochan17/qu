# Japanese Transcript Polisher

日本語の文字起こしを、教材制作で扱いやすいMarkdownに整えるための手順。

## 原則

- 原文の意味と説明順序を保つ。内容を勝手に追加しない。
- YouTube等の第三者コンテンツは、公開教材へそのまま転用しない。参照資料として整え、教材化時は要約・再構成・出典管理を行う。
- まず原文ファイルを残し、整形版を `*.cleaned.md` または `drafts/` に出す。ユーザーが明示した場合だけ原文を上書きする。
- 数式、専門用語、試験用語は正確性を優先する。怪しい箇所は無理に断定せず、あとで確認できるよう表現を平易にする。

## 標準ワークフロー

1. 対象ファイルのサイズ、行数、先頭部分を確認する。
2. 文字起こしが1行巨大テキストの場合、`scripts/clean_transcript.py` で一次整形する。
3. 一次整形後、章見出し・専門用語・明らかな誤変換を確認する。
4. 教材下書きにする場合は、別途「学習目標」「要点」「確認問題」へ再構成する。参照資料の整形だけなら原文順の本文に留める。

## 整形ルール

- 不要な空白を詰め、日本語として自然な句読点を入れる。
- 「えー」「あの」「まあ」「ですね」など、意味を持たないフィラーは削る。
- 「こんにちは。本日は...」「それでは早速...」などの講義区切りは、見出し候補にする。
- 「動画を止めて考えてみてください」など、動画視聴前提の指示は、参照資料では自然な文章へ寄せるか削る。
- 「奇数変換」のようなASR誤変換は、文脈から明らかな場合のみ「基数変換」に直す。
- 用語は基本情報技術者試験で一般的な表記へ寄せる。

## よくある補正

- `奇数` → 文脈がN進数なら `基数`
- `進数` → 文脈に応じて `10進数` または `N進数`
- `shif DIS C` / `shif デコード` → `Shift_JIS`
- `ユニコード` → `Unicode`
- `売上 減価` → `売上原価`
- `原価 償却` → `減価償却`
- `対応 年数` → `耐用年数`
- `警報` → 法規の文脈では `刑法`
- `式 命令` → `指揮命令`
- `受け 追い` → `請負`
- `企画` → 標準化の文脈では `規格`

## 付属スクリプト

`scripts/clean_transcript.py`:

```bash
python3 .claude/skills/japanese-transcript-polisher/scripts/clean_transcript.py input.md output.cleaned.md
```

これは一次整形用。最終教材化では、生成されたMarkdownを読み、章単位で自然さと正確性を追加確認する。
